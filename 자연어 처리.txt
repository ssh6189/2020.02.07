자연어 처리의 어려움

언어, 상황, 환경, 지각 지식의 학습 및 표현의 복잡함
Rule 기반만으로는 무리인가?
DNN은 분산 표현의 장점으로 인해, 모호하지만, 풍부한 정보를 얻을 수 있다.
-> 단어의 벡터화로부터 시작



언어 데이터의 특성
Audio, Image, Text
연속, 연속, 불연속

Text는 크기라는것이 없다. 단어들읠 무한대의 조합이다.



언어데이터의 특성
불연속적인 심볼의 계열(Sequence)
계열 길이가 샘플에 따라 다르다.

계층적 재귀적인 구조를 지닌다.

(문자, 단어, 구, 문)



1. 계열 -> 계열 (Sequence : 계열)
형태소 해석
기계번역, 자동요약
질의응답, 챗봇

2. 계열 -> 트리구조
구문 해석

3. 계열 -> 그래프 구조
의미 해석



단어의 국소표현

고양이 [1, 0, 0, 0, 0]
개 [0, 1, 0, 0, 0]
사람 [0, 0, 1, 0, 0]

이것만 가지고는 단어의 의미를 전혀 알 수 없다. -> 단어의 의미를 파악하는 벡터를 갖고 싶다.



분포가설

비슷한 문맥을 가진 단어는 비슷한 의미를 갖는다. : 현대의 통계적 자연어 처리에서 획기적인 발상
(전반적으로는 맞으나, 100%는 아니다.)



분포 가설에 기반한 방법은 크게 2종류로 나눈다.

Count-based methods
예 : SVD (LSA), HAL, etc
단어, 문맥, 출현 횟수를 세는 방법

Predictive methods
예 : NPLM, word2vec, etc
단어에서 문맥 또는 문맥에서 단어를 예측하는 방법



규칙을 이용한 학습

0 : 가위, 1 : 바위, 2 : 보

9가지 규칙을 가지고, 승, 무, 패를 결정

이런게 규칙 기반 모델링

(my - computer + 3) % 3
#음수가 되는것을 방지하고자 -3
0 : 비김
1 : 이김
2 : 짐



##########참고##########
(-1 + 3) % 3 = 2

(1 + 3) % 3 = 1
(1 + 6) % 3 = 1

이와 같은 원리를 이용 뒤에 3을 더하는건, 영향을 주지 않는다.
